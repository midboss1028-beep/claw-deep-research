# Open Deep Research

> 拒绝表面文章。用迭代搜索 + 递归深挖，像人类研究者一样层层深入，产出真正有深度的分析报告。

## 这是什么

一个给 AI Agent 用的**深度研究工作流**。把"研究"这件事从"搜一下然后总结"升级为：

```
一层搜索 → 提炼认知 → 发现盲区 → 继续深挖 → 合并输出
```

灵感来自 [dzhng/deep-research](https://github.com/dzhng/deep-research)（17.8k ⭐）的核心算法，但：
- **不需要额外的 API Key** — 复用 OpenClaw 已有的 `web_search` + `web_fetch`
- **不需要额外部署** — 一份 SKILL.md，放到 skills 目录就能用
- **模型无关** — 你用什么 LLM 就跑什么模型，当前对话的模型就是引擎

## 前置条件

任何正常运行且配置了 `web_search` 提供商的 OpenClaw 实例：

- `web_search` — 已配置（MiniMax / Tavily / Brave / Perplexity / Google 均可）
- `web_fetch` — 可用（默认开启）
- 以上，没有其他依赖。

**不挑搜索提供商，不挑 LLM，不挑操作系统。**

## 核心算法

**广度搜索 × 深度递归** 双层结构：

```
参数：breadth=4（每层搜索数）, depth=2（递归层数）
                           ┌─ 搜索1 → 提炼认知 → 递归(广度2,深度1)
        用户问题 ─→ 生成4路搜索 ── 搜索2 → 提炼认知 → 递归(广度2,深度1)
                           ├─ 搜索3 → 提炼认知 → 递归(广度2,深度1)
                           └─ 搜索4 → 提炼认知 → 递归(广度2,深度1)
                                            │
                                            ↓
                                   合并所有认知 → 最终报告
```

- **广度（breadth）：** 每层生成多少条搜索方向，默认 4，子层自动减半为 2
- **深度（depth）：** 递归层数，默认 2（首层 4 条 + 下层 8 条，约 12 次搜索）
- **认知池（learnings）：** 全局累积、去重，贯穿整个研究过程
- **来源池（visitedUrls）：** 全局累积，最终写入报告附录

## 完整工作流

### 阶段 0：解析输入

用户触发后（显式：`/deep <topic>` | 隐式：「帮我深度研究一下 XXX」），提取：

- **主题** — 研究的核心对象
- **范围** — 关注的维度（技术/商业/政策/历史/竞品/学术……）
- **格式偏好** — 报告/对比/时间线/列表/问答……（默认结构化报告）
- **广度/深度** — 如无指定，默认 breadth=4, depth=2

**输出：** 研究计划（大纲 + 参数）

### 阶段 1：生成搜索查询

根据当前状态生成 N 条搜索查询：

- **输入：** 原始问题 + 已有认知（learnings，首轮为空）
- **输出：** 每条包含 query（搜索词） + researchGoal（研究目标 + 后续方向）
- **数量：** 当前层 breadth 值（首层默认 4，子层减半）

researchGoal 要具体：不仅说明这次搜索要发现什么，还要说明找到结果后下一步往哪个方向深挖。

### 阶段 2：执行搜索并提炼认知

对每条查询：

1. 调用 `web_search(query, count=5)` 获取搜索结果
2. 如有必要，调用 `web_fetch(url)` 获取详情页完整内容
3. 根据搜索结果，用 LLM 提炼：

**关键认知（learnings）** — 要求：
- 包含具体实体（人、公司、产品、机构、地名等）
- 包含具体数字/指标/日期
- 每一条是一个独立的信息点，不重复
- 精炼、信息密度高

**后续方向（followUpQuestions）** — 要求：
- 基于当前认知自然延伸
- 指向尚未覆盖的维度或矛盾点
- 用于下一层递归的输入

### 阶段 3：递归深挖

当 depth > 0 时：

1. 合并当前层全部认知到全局认知池
2. 合并当前层全部 URL 到全局 URL 池
3. 递归调用自身，参数更新：
   - breadth = max(2, Math.ceil(currentBreadth / 2))
   - depth = depth - 1
   - query = 上一轮的 researchGoal + followUpQuestions
   - learnings = 全局认知池（去重）
   - visitedUrls = 全局 URL 池（去重）

当 depth = 0 时停止递归，进入报告生成阶段。

**注意：** 如果某条搜索失败或超时，跳过该条继续执行，不影响整体流程。

### 阶段 4：生成报告

基于全局认知池生成结构化报告：

```markdown
# [主题] — 深度研究报告

## 摘要
300-500 字综述，涵盖所有关键发现和结论

## 背景
主题的背景、上下文、研究范围和方法说明

## 核心发现
### 子主题一
发现 + 数据 + 分析 + 来源标注

### 子主题二
发现 + 数据 + 分析 + 来源标注

...

## 综合洞察
跨维度的趋势判断、矛盾点分析、独特见解

## 结论
核心结论 + 局限 + 进一步研究方向

## 来源
- [标题](url)
- [标题](url)
...
```

**报告产出格式：** Markdown（中文首选，按用户输入语言适配）

## 交互反馈

执行过程中**全程透明**：

1. **起步 ⏤** 输出研究计划：「🔬 开始深度研究：[主题]，广度=4，深度=2，预计搜索约12次」
2. **每层完成 ⏤** 输出进展：「🔬 [深度 2/2] 本轮搜索完成，累计获取 15 条认知」
3. **完成 ⏤** 输出完整报告 + 附带来源清单

## 自适应调整

| 输入特征 | 调整策略 |
|----------|---------|
| 话题狭窄：如「XX公司Q3营收」 | breadth=2, depth=1 |
| 话题宽泛：如「AI 芯片竞争格局」 | breadth=6, depth=2 |
| 话题极端宽泛：如「人类文明史」 | breadth=4, depth=3 |
| 用户要求"简要" | breadth=3, depth=1 |
| 用户要求"详尽" | breadth=5, depth=3 |
| 搜索提供商并发限制较低 | 自动降低 breadth，避免超时 |

## 触发规则

### 显式触发
- 用户说 `/deep <topic>`
- 用户说 `/deep research <topic>`

### 隐式触发（根据 description 自动匹配）
- 用户说「帮我深度研究一下 X」
- 用户说「深度调研 X」
- 用户说「深挖一下 X」
- 用户说「出一份关于 X 的研究报告」
- 用户说「做个关于 X 的深度分析」

### 不触发场景
- 随手查事实：「帮我查一下 X」
- 简单问答：「X 是什么」
- 单次搜索请求：「搜一下 X」
- 日常工作流触发

## 无障碍适配

- 中文用户 → 中文搜索词、中文报告
- 英文用户 → 英文搜索词、英文报告
- 混合输入 → 按用户输入语言自动适配

## 常见反模式

| ❌ 反模式 | ✅ 正确做法 |
|----------|------------|
| 一次性搜很多方向但不递归 | 宁可减少广度，保证深度 |
| 递归时重复搜索已有内容 | 每轮先 review 已有认知，精准定位盲区 |
| 报告只列点没有分析 | 每个子主题必须有「分析」「洞察」段落 |
| 忽略来源引用 | 每个关键数据标注出处来源 URL |
| 报告写成 bullet list | 完整段落 + 分析性语言 |
| 遇到失败直接停掉整个流程 | 跳过失败分支，其他分支继续执行 |
